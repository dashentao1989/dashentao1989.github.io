proguard 那些事

With each build, ProGuard outputs the following files:

dump.txt

  • 描述apk所有文件的内部结构
    Describes the internal structure of all the class files in the APK.

mapping.txt

  • 提供了一个源文件与混淆文件的对照
    Provides a translation between the original and obfuscated class, method, and field names.

seeds.txt

  • 列出没有被混淆的类和成员变量
    Lists the classes and members that were not obfuscated.

usage.txt

  • 列出apk中被移除的代码 Lists the code that was removed from the APK.*
  • 这些文件保存在/build/outputs/mapping/release下面
    These files are saved at /build/outputs/mapping/release/.
  • 要值得注意的是在每次通过proguard打release包时mapping.txt文件都会被重写， 所以你必须仔细的保存一份复制在打一个新的而release包时，通过在打release包时保留一个mapping.txt文件，这样的话当用户在使用老版本app时提出提出混淆的错误信息是你能根据mapping.txt对照关系进行相应准确的deug.

Be aware that the mapping.txt file is overwritten every time you create a release build with ProGuard, so you must carefully save a copy each time you publish a new release. By retaining a copy of the mapping.txt file for each release build, you’ll be able to debug a problem if a user submits an obfuscated stack trace from an older version of your app.

Vim使用心得

从今天开始学习Vim不要阻止我

Better.Stronger.Faster

  • i 编辑
  • :wq ZZ :x 保存退出
  • :q! 退出不保存
  • yy 复制
  • p 粘贴
  • :q 退出不保存
  • v 可视化
  • hjkl 光标左上右下移动
  • 0 光标移动到该行开头
  • $ 光标移动到该行末尾
  • ^ 光标移动到该行第一个不是blank的地方
  • g_ 光标移动到该行最后一个不是blank的地方
  • w 移动到下个单词的开头
  • e 移动到下个单词的结尾
  • :N 移动到第N行
  • :set number 显示行号
  • :set nonumber 不显示行号
  • . 执行上次操作
  • .N 执行上次操作N次
  • dd 删除改行
  • Ndd 删除N行
  • o 该行前面插入新行
  • O 该行后面插入新行
  • N 重复执行command命令N次
  • % 匹配括号移动，包括 (, {, [. （陈皓注：你需要把光标先移到括号上）
  • *和#: 匹配光标当前所在的单词，移动光标到下一个（或上一个）匹配单词
  • v 可视化
  • fa 到下个为a的字符处，你也可以fs到下一个为s的字符
  • 3fa 在当前行查找第三个出现的a
  • F和T 和f和t一样，只不过是相反的方向
  • /pattern 寻找pattern匹配项
  • gU 变大写
  • gu 变小写
  • y2/foo 拷贝2个“foo”之间的字符串
  • 或是ctrl-n 自动提示功能
