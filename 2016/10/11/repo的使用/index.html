repo的使用

发表于 2016-10-11   |  
repo简介：

repo是Android项目团队为了方便管理Android项目及组成Android项目的几百个项目的Git库而开发的一套Python脚本。

Android使用Git作为代码管理工具，开发了Gerrit进行代码审核以便更好地对代码进行管理，还开发了 Repo 命令行工具，对Git部分命令进行了封装，将多个Git库有效地管理起来。

构成repo的一系列Python脚本自己也组成了一个git库，这个库用于管理repo自身的开发和演进。

构成repo的这些Python脚本，本质上还是调用了Git的各项命令来实现相应的管理功能的。

repo管理所设计的三类库
  • repo库

这个是管理组成repo的Python脚本的Git库，就是负责repo相关的一组脚本的演进和更新。

  • Manifest库

这个库每个项目都会有。这是管理一个项目的所有相关库及历史分支的库,其中有许多个Manifest文件，每个Manifest文件应该是描述了一个项目包含的所有子库的信息。

  • project库

这是真正存储项目的源代码的库，可能有一组许多个Git库,每个Git库分别管理自己库的演进。

关于Manifest库中的清单文件的进一步说明

一个清单库可以包含多个清单文件和多个分支，每个清单文件和分支都有对应的版本。

清单文件以xml格式组织。包含如下主要元素：

  • remote元素

定义远程版本库相关的信息，会给出远程Git库的访问路径。

  • default元素

设置各个项目默认的远程版本库，默认的分支,各个project元素中，还可以定义自己的remote和revision，以覆盖默认设置。

  • project元素

用于定义一个项目，path属性表示在工作区克隆的位置，name属性表示该项目的远程版本库的相对路径。

  • copyfile元素

定义了项目克隆后的一个附加动作，从源位置向目标位置拷贝文件。

使用repo的获取代码的流程
  • 通过一个初始的repo库版本，把repo自身更新到最新状态
  • 通过repo库把某个项目相关的Manifest库取下来，并指定使用Manifest库中的某个Manifest文件来获取项目库。

前两步在执行repo init命令时执行，在使用repo init命令取manifest库时，会自动更新repo库。

  • 获取项目库

执行repo sync命令取项目库。

repo的常用命令：

repo init

  • -u
  • -m
  • -b
  • –repo-url
  • –no-repo-verify

repo sync

用于参照清单文件克隆并同步版本库。
可以使用repo sync project_name的形式只克隆某个项目。。

如果某个项目版本库不存在，则执行repo sync命令相当于git clone。 如果库已经存在，则相当于执行下面两条命令：

git remote update - 相当于对每一个remote源执行了fetch操作
git rebasee origin/branch - 针对当前分支的跟踪分支执行rebase操作。

repo start

创建并切换分支。

刚克隆下来的代码是没有分支的，repo start实际是对git checkout -b命令的封装。
为指定的项目或所有的项目（若使用-all），以清单文件中为设定的分支，创建特定的分支。
这条指令与git checkout -b 还是有很大区别的。
git checkout -b 是在当前所在的分支的基础上创建特性分支。
而repo start 是在清单文件设定的分支的基础上创建特性分支。

假设清单文件中设定的分支是gingerbread-exdroid-stable，那么执行以上指令就是对所有项目，在gingerbread-exdroid-stable的基础上创建特性分支stable

假设清单文件中设定的分支是gingerbread-exdroid-stable，那么执行以上指令就是对platform/build、platform/bionic项目，在gingerbread-exdroid-stable的基础上创建特性分支stable。

repo checkout

切换分支。 […]

实际上是对git checkout命令的封装，但不能带-b参数，所以不能用此命令来创建特性分支。

repo diff

查看工作区文件差异。

实际是对git diff命令的封装，用于分别显示各个项目工作区下的文件差异。

repo stage

把文件添加到index表中。

实际上是对git add –interactive命令的封装，用于挑选各个项目中的改动以加入暂存区。

-i表示git add –interactive命令中的–interactive，给出一个界面供用户选择。

repo prune

删除已经合并分支

实际上是对git branch -d 命令的封装，该命令用于扫描项目的各个分支，并删除已经合并的分支。

repo abandon

删除指定分支。

实际是对git branch -D命令的封装。

repo status

查看文件状态。

repo remote
设置远程仓库。

repo push
向服务器提交代码。

repo forall

迭代器，可以在所有指定的项目中执行同一个shell指令。

选项：

  • -c 后面所带的参数是shell指令
  • -p 在shell指令输出之前列出项目名称
  • -v 列出执行shell指令输出的错误信息

附加环境变量：

  • REPO_PROJECT 指定项目的名称
  • REPO_PATH 指定项目在工作区的相对路径
  • REPO_REMOTE 指定项目远程仓库的名称
  • REPO_LREV 指定项目最后一次提交服务器仓库对应的哈希值
  • REPO_RREV 指定项目在克隆时的指定分支，manifest里的revision属性
    如果-c后面所带的shell指令中有上述环境变量，则需要用单引号把shell指令括起来
